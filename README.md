## 📖 專案簡介

**呼嚕嚕小鎮** 是一款結合「遊戲化激勵」與「實用管理」的班級經營工具。由於深知教學現場的痛點，因此捨棄了繁瑣的註冊流程，採用 **本地優先 (Local-First)** 架構，讓老師打開瀏覽器就能立即使用，資料完全掌握在自己手中。

無論是電子白板上的全班展示，還是老師手持平板的行間巡視，呼嚕嚕小鎮都能完美適配。

---

## 🌟 核心亮點

### 🖥️ 雙模式完美適配
* **電子白板原生模式 (1920 x 1024)**：專為大螢幕設計，單頁呈現所有資訊，無需捲動，讓全班學生一目瞭然。
* **行動巡堂模式 (RWD)**：支援手機與平板，介面自動轉為垂直流式佈局。老師走到哪，管理就到哪。

### ✨ 獨家任務追蹤系統
超越傳統的「已交/未交」，我們設計了更符合現場需求的狀態機：
* **⚡ 智慧發布**：輸入「國習」、「水彩」，系統自動配對可愛圖示。
* **📊 多元狀態**：
    * 🟢 **準時**：計入達成率，獲得完整獎勵。
    * 🟡 **遲交/補交**：雖然晚了，但還是完成了！給予鼓勵。
    * 🔴 **未交**：視覺化紅燈警示，掌握落後名單。
    * ☕ **請假** / ⛔ **免交**：人性化設計，不計入分母，照顧特殊需求。

### 🎨 溫暖療癒體驗
* **30 種動物村民**：豐富的 Emoji 動物頭像庫（🦓 🦒 🐘 🦔...），每位學生都是獨一無二的村民。
* **榮譽激勵**：小隊全員完成任務時，區塊會亮起 **金框特效** 🏆。
* **擬真布告欄**：軟木塞風格的班級公告區，重要事項隨手貼。

## ☁️ 雲端備份操作教學 (Google Apps Script)

本系統支援將資料備份至您的 **Google 雲端試算表 (Google Sheets)**，實現跨裝置同步（例如：在學校電腦操作，回家用筆電繼續編輯）。

### 步驟 1：建立雲端資料庫
1.  登入您的 Google 帳號，前往 **Google Drive**。
2.  建立一個新的 **Google 試算表**，命名為「呼嚕嚕小鎮資料庫」或您喜歡的名稱。
3.  點擊上方選單的 **「擴充功能」** -> **「Apps Script」**。

### 步驟 2：部署後端程式
1.  在開啟的 Apps Script 編輯器中，刪除原本的 `myFunction` 代碼。
2.  複製本專案提供的 `Code.gs` 檔案內容，貼入編輯器中。
3.  **⚠️ 重要安全設定**：
    * 找到程式碼最上方的 `const TOKEN = "meow1234";`。
    * 請務必將 `"meow1234"` 修改為您自訂的密碼（例如 `"mySecretClass808"`），以防止他人誤連您的資料庫。
4.  點擊上方 **「部署」** -> **「新增部署作業」**。
5.  點擊左側齒輪圖示，選擇 **「網頁應用程式」**。
6.  設定如下（**重要！**）：
    * **執行身分**：`我 (Me)`
    * **誰可以存取**：`任何人 (Anyone)` *(這是為了讓 App 能寫入資料)*
7.  點擊「部署」，依指示授權存取權限。

### 步驟 3：連結系統
1.  部署成功後，複製畫面上的 **「網頁應用程式網址 (Web App URL)」**。
2.  回到呼嚕嚕小鎮，點擊右上角 **「設定 (齒輪)」**。
3.  在 **「雲端備份中心」** 區塊：
    * **GAS 部署網址**：貼上剛剛複製的網址。
    * **驗證 Token**：輸入您在步驟 2 設定的密碼（預設為 `meow1234`，強烈建議修改）。
4.  點擊 **「☁️ 雲端上傳」** 測試連線，若顯示成功即完成設定！

## 📦 快速開始

### 開發環境建置

```bash
# 1. 下載專案
git clone [https://github.com/your-username/purr-purr-town.git](https://github.com/your-username/purr-purr-town.git)

# 2. 安裝依賴
npm install

# 3. 啟動開發伺服器
npm run dev
```

### 部署至 GitHub Pages

```bash
# 建置專案
npm run build

# 部署 (需先設定好 gh-pages)
npm run deploy
```
